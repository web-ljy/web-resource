什么是http?

超文本传输协议 hypertext transport protocol



URL:指的是统一资源定位符

port:http默认port为80    https默认为443

location.hostname   // 指获取网站的域名



网络的七层协议：

1、物理层 --> 网卡

2、链路层 --> 网卡驱动

3、网络层 --> ip

4、传输层 --> tcp/udp 传输协议

5、回话层 --> 保持会话/断开会话

6、加密层 --> 加密解密、对称加密/非对称加密

7、应用层 --> http https(加密过的http) ftp smtp... 等



http1.1 和 http2.0 的区别？

1、多路复用（减少tcp请求，把多个合并成一个请求）

2、首部压缩

3、服务器推送


页面的同步请求和异步请求：

1、同步：当前页面内的任何一个操作都需要重新刷新页面。

2、异步：不需要每次刷新页面，只需要加载指定的内容即可。


http method:

1、get  --> 从服务器获取资源

2、post --> 向服务器发送资源

3、put  --> 向服务器推送资源

4、delete -->  告诉服务器删除某个资源

5、head --> 告诉服务器返回数据时不需要返回响应主体。只返回起始行和首部即可。（获取服务器事件、下载之前获取下载文件名和文件大小）

6、options --> 检测服务器支持哪些http方法


GET方法的特点：

一、没有请求主体

二、会把查询条件格式化为URL的格式拼接到url后面

三、有大小限制；因为浏览器对url的长度有限制。所以造成了get请求有大小限制。

四、明文发送；不安全；安全性比较差。

五、浏览器默认全是get请求


POST方法的特点：

一、有请求主体

二、没有大小限制；因为它把数据全部放到请求主体之中；而浏览器对请求主体是没有大小限制的。

三、不是明文发送；安全性较强。

四、不会被浏览器缓存


http状态码：

标识server处理的状态

1开头的：http状态初始化 

        101

2开头的：客户端成功

        200 OK 成功

        202 accepted 接受

3开头的：重定向

        301 永久转移
 
        302 临时转移

        301和302都有一个共同的特点：从站点a跳转到站点b

        304 未修改、缓存、注意304的原理是什么？

4开头的：客户端错误(大部分前端错误)

        400 错误的请求 端口不正确或者参数不正确

        401 未认证 没有登录信息

        403 禁止访问 没有权限

        404 未找到 该资源未找到

5开头的：服务器错误

        500 内部服务器错误

        503 连接数过多
